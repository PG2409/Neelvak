# Neelvak: The Enterprise Intelligence & Execution Operating System

Neelvak is a production-grade, model-independent Enterprise Intelligence and Execution Operating System. Designed as a resilient, auto-healing AI microkernel, Neelvak completely decouples core organizational logic from underlying foundation models. Rather than serving as an AI wrapper, chatbot, or simple agent framework, Neelvak operates as a decentralized gateway that orchestrates intent compiler pipelines, enforces strict security policies, manages persistent memory layers, and performs dynamic cost optimization across cloud, local, and hybrid computing substrates. By establishing an open execution standard via dual-transport protocols (HTTP + MCP), Neelvak guarantees that an organization's intelligence remains private, persistent, and decoupled from vendor lock-in.

## 2. Intent-Based Cognitive Engine

Neelvak does not think in terms of static prompts, fragile agent scripts, or specific model parameters. Instead, it evaluates execution along a structured, declarative intent pipeline:

```
                    [ USER ]
                       │
                       ▼
                   [ INTENT ]
                       │
                       ▼
                [ ORGANIZATION ]
                       │
                       ▼
                  [ KNOWLEDGE ]
                       │
                       ▼
                 [ CAPABILITY ]
                       │
                       ▼
                  [ WORKFLOW ]
                       │
                       ▼
                  [ EXECUTION ]
                       │
                       ▼
                   [ MEMORY ]
                       │
                       ▼
                   [ ACTION ]
                       │
                       ▼
                   [ OUTPUT ]
```

*   **Intents:** What the user or system is trying to accomplish semantically.
*   **Capabilities:** The abstract skills required (e.g., Reasoning, Coding, Analysis).
*   **Organizational Intelligence:** The domain rules, policy bounds, and data access limits.
*   **Execution Plans:** Declarative DAGs of work generated dynamically by the compiler.
*   **Workflows:** Topological execution layers resolving dependencies sequentially or in parallel.
*   **Actions:** Actual executions occurring against tool interfaces or local environments.

## 7. System Capabilities

### Intent Gateway & 3-Tier Routing
*   **Problem Solved:** Over-provisioning complex LLMs for simple tasks (e.g., using GPT-4 for "hello").
*   **Why It Exists:** To drastically reduce latency and operational API costs.
*   **How It Works:** Compiles prompts against a semantic matrix. Tier 1 (definitions/greetings) executes instantly on low-resource local threads. Tier 2 (creative generation) targets intermediate models. Tier 3 (analytical operations) spawns multi-model validation pipelines (e.g., executing Llama-3.3-70b alongside Qwen-32b as a verification partner).

### Policy Validation & Security Ring
*   **Problem Solved:** Sensitive corporate data (API keys, PII, intellectual property) leaking to cloud providers.
*   **Why It Exists:** Complete compliance and security boundaries at runtime.
*   **How It Works:** Interceptscompiled plans prior to execution. Runs deterministic regex and semantic scanners to block unauthorized outbound payloads.

### Dual-Transport Subsystem
*   **Problem Solved:** Coupling business logic directly to frontend web applications.
*   **Why It Exists:** To allow the same engine to power a user-facing dashboard or interact natively with developer tools (Cursor, Claude Desktop).
*   **How It Works:** Boots standard HTTP/ASGI servers for web telemetry, or instantly shifts into Model Context Protocol (MCP) mode via standard I/O (STDIO) to act as a plugin server.

## 8. Runtime & Configuration

### Prerequisites & Setup
Install the core OS requirements:
```bash
pip install -r requirements.txt
```

### Launch Web Telemetry Dashboard
Boots the ASGI interface on port 8000:
```bash
python main.py
```

### Launch Model Context Protocol (MCP) Server
Boots the engine as a stateless STDIO server:
```bash
python main.py --mcp
```

### IDE Configuration (Cursor / Claude Desktop)
To mount Neelvak as a native system capability within your IDE, add the following to your MCP settings file:
```json
{
  "mcpServers": {
    "neelvak-orchestrator": {
      "command": "python",
      "args": [
        "c:/neelvak/main.py",
        "--mcp"
      ]
    }
  }
}
```
